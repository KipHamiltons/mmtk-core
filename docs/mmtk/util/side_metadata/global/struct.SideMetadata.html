    size: usize
) -> Result<()>[src]

Tries to map the required metadata address range, without reserving swap-space/physical memory for it. This will make sure the address range is exclusive to the caller. This should be called at chunk granularity.

NOTE: Accessing addresses in this range will produce a segmentation fault if swap-space is not mapped using the try_map_metadata_space function.

pub fn ensure_unmap_metadata_space(&self, start: Address, size: usize)[src]

Unmap the corresponding metadata space or panic.

Note-1: This function is only used for test and debug right now.

Note-2: This function uses munmap() which works at page granularity. If the corresponding metadata space's size is not a multiple of page size, the actual unmapped space will be bigger than what you specify.
